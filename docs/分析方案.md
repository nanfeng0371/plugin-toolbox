## Sheet2：课节汇总分析

### 核心思路
- 从Sheet1数据中，按「本课节名称」分组
- 每个独立课节一块汇总区域
- 自动统计：这个课节整体表现如何、哪些学生有问题、哪个知识点全班最弱

### 结构：每个课节一个区块

#### 区块标题行
```
━━━ 【读】文学感知"超频"模式：这些分别放过 ━━━
```

#### 子表1：课节概览（固定6行）

| 行 | 指标 | 值 |
|----|------|-----|
| 1 | 学生总数 | XX人 |
| 2 | 平均回答率 | XX% |
| 3 | 平均掌握度分布 | A: X人 / B+: X人 / B: X人 / C: X人 |
| 4 | 敷衍预警人数（<50%） | XX人 |
| 5 | 严重敷衍人数（<30%） | XX人 |
| 6 | 平均练习正确率 | XX% |

#### 子表2：该课节学生排名（每人一行）

| 列 | 内容 |
|----|------|
| A | 排名 |
| B | 姓名 |
| C | 回答率% |
| D | 掌握度 |
| E | 练习正确率% |
| F | 综合标签 |
| G | 一句话诊断 |

→ 按**回答率升序**排列（最敷衍的在最上面）

#### 子表3：该课节知识点表现（每知识点一行）

| 列 | 内容 |
|----|------|
| A | 知识点名称 |
| B | 出现人次（多少学生有这个点） |
| C | 平均掌握度 |
| D | 平均回答率 |
| E | 平均首次答对率 |
| F | 平均练习正确率 |
| G | 未完成人数 |
| H | **薄弱判定** | 全班普遍弱 / 部分薄弱 / 正常 |

#### 子表4：该课节问题学生名单（仅列出⚠️及以上问题的）

| 列 | 内容 |
|----|------|
| A | 姓名 |
| B | 问题类型 | （敷衍/未掌握/两者都有）|
| C | 关键指标 | （如"回答率22%"）|
| D | 建议动作 | （联系家长/调整教学/重点辅导）|

## 执行流程更新

```
1. 读取输入文件 → 200个report token
2. 循环请求接口 → 提取每个学生的全部数据
3. 写入 Sheet1（学生明细，96列 × 200行）
4. 按"本课节名称"分组
5. 对每组计算汇总统计
6. 写入 Sheet2（课节汇总，动态生成N个区块）
7. 输出到 D:\Claw\学习报告获取\output\学习报告分析_YYYYMMDD.xlsx
8. 预计耗时 5-8分钟
```
