## 一、产品目标

让辅导老师的每日工作**可视化、可追踪、可管理**。

- **辅导老师视角**：一眼看清今天所有学生的状态，逐项完成并打勾
- **管理视角**：实时看到每个辅导老师的工作完成情况，数据驱动管理决策

---

## 二、核心流程

```
① 辅导老师绑定学情表（一次性）
        ↓
② 插件自动获取今日排课 + 学生学情
        ↓
③ 系统按规则自动分类学生（7类）
        ↓
④ 辅导老师逐类完成动作，打勾标记
        ↓
⑤ 数据上传云端
        ↓
⑥ 管理在 EduFlow 看板查看团队数据
```

---

## 三、学生分类逻辑（核心）

### 3.1 时间状态定义

| 时间段 | 状态 |
|--------|------|
| 当前时间 < 排课开始时间 | 今天有课 - 未上课 |
| 排课开始时间 ≤ 当前时间 < 排课结束时间 | 今天有课 - 正在上课 |
| 当前时间 ≥ 排课结束时间 | 进入「已下课」判断 |

### 3.2 已下课子分类

| 分类 | 判断条件 | 辅导动作 |
|------|---------|---------|
| 未生成学习报告 | 课后无报告数据 | 电话联系重约课 + 打勾确认 + 系统校验是否已重约 |
| 优秀 | 报告四维评价 = ⭐优秀 | 向学员要笔记 / 抽查 / 完整复述 |
| 敷衍但会 | 报告四维评价 = 需关注/认真 | 向学员抽查 / 完整复述 |
| 敷衍 | 报告四维评价 = 敷衍预警/敷衍+未掌握 | 电话联系家长 |

### 3.3 无课学生

| 分类 | 判断条件 | 辅导动作 |
|------|---------|---------|
| 今天没课 | 今日排课列表中没有该学生 | 单独私聊布置题目 |

## 四、功能模块

### Module A：学情表绑定（插件侧）

**触发时机**：首次使用，或学情表有变更时

**输入方式**：辅导老师在插件侧边栏「每日看板」Tab 中粘贴学情表

**数据格式**（Tab 分隔）：
```
学生ID    学生姓名    手机号    年级    中心    其他字段...
```

**辅导老师标识**：从爱芯后台页面自动提取当前登录的辅导老师姓名作为 `teacherId`（目前无重名，无需额外账号体系）

**匹配逻辑**：以学生 ID 为 key，与排课 API 返回的 `studentId` 字段匹配

**存储**：`chrome.storage.local`（本地缓存），每次打勾/刷新后实时同步到云端

### Module C：数据获取与更新（插件侧）

**数据来源**：

| 数据 | 来源 API | 获取方式 |
|------|---------|---------|
| 今日排课 | 热力图模块已有排期 API | 复用，按当天日期筛选 |
| 学生学情 | 学情表（本地） | 本地匹配 |
| 学习报告 | 学习报告插件已有 API | 复用，课后触发获取 |
| 上课状态 | 爱芯监控 API（待验证） | 轮询或手动刷新 |

**刷新逻辑**：
- 手动点击「刷新数据」按钮（全量刷新排课 + 报告数据）
- 「已下课-未生成报告」状态：课后延迟 30 分钟再检查报告是否生成
- 数据变化后**实时同步到云端**，管理看板立即可见

## 六、待确认问题（已全部确认）

| # | 问题 | 结论 |
|---|------|------|
| 1 | 辅导老师的「teacherId」从哪来？ | ✅ **用辅导老师姓名**。目前无重名，插件从后台页面自动提取当前登录的辅导老师姓名作为唯一标识 |
| 2 | 排期 API 能否按「辅导老师」筛选？ | ✅ **不需要筛选**。每个辅导老师登录的是自己的后台，排期 API 返回的就是该老师自己的数据，无需额外过滤 |
| 3 | 「已下课-未生成报告」状态允许延迟多久？ | ✅ **延迟 30 分钟**。课后 30 分钟内未生成报告属于正常等待，超过 30 分钟仍未生成则触发提醒 |
| 4 | 管理看板的实时性要求？ | ✅ **实时同步**。辅导老师插件侧数据变化（打勾/刷新）→ 立即上传云端 → 管理看板直接读取云端实时数据，无需 WebSocket |
| 5 | 重约校验逻辑优先级？ | ✅ 第一期先不做自动校验，只保留「已重约」勾选项，后续验证 API 可行性后再加入 |

## 七、技术风险

| 风险 | 影响 | 缓解方案 | 状态 |
|------|------|---------|:--:|
| 爱芯监控 API 字段未验证 | 无法实时判断「正在上课」的学生在线/离线状态 | 第一期用「时间在课表中」代替，不区分在线/离线 | 🟡 |
| ~~排期 API 不支持按辅导老师筛选~~ | ~~数据量过大~~ | ✅ 已确认：每个辅导老师后台只返回自己的数据 | ✅ |
| CloudBase 读写权限 | 辅导老师只能写自己的数据，管理能读全量 | 在 CloudBase 配置安全规则，按 teacherName 隔离 | 🟡 |
| 学情表格式不统一 | 不同辅导老师粘贴的格式可能有差异 | 提供模板下载，前端格式校验提示 | 🟡 |
| 辅导老师重名 | 未来可能出现同名辅导老师 | 当前无重名，预留 `teacherId` 字段，未来可扩展为「姓名+工号」 | 🟢 |

## 八、一期范围建议

**MVP（最小可用）**：

1. ✅ 学情表绑定（粘贴 + ID 匹配）
2. ✅ 7类分类（基于时间判断，报告数据可选）
3. ✅ 打勾标记（本地保存）
4. ✅ 云端上传（手动触发）
5. ✅ EduFlow 管理看板（只读，显示团队汇总）

**一期不做**：
- ❌ 实时上课状态（爱芯 API 未验证）
- ❌ 重约校验（用户已确认排后）
- ❌ 自动刷新状态（手动刷新即可）
- ❌ 消息提醒 / WebSocket
