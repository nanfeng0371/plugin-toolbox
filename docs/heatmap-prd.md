**文档版本**: v1.0  
**撰写人**: 许清楚（Xu）- 产品经理  
**日期**: 2026-06-06  
**状态**: 待评审

---

## 一、产品目标

为课程顾问和教学管理人员提供可视化的课程排期分析工具，通过热力图直观展示排课密度，快速识别排课异常，并支持批量导出学生课程日历，提升排课管理效率和质量。

---

## 二、用户故事

| # | 作为… | 我希望… | 以便… |
|---|-------|---------|--------|
| 1 | 课程顾问 | 看到每周各时段排课密度的热力图 | 快速识别热门时段和空档时段，优化排课策略 |
| 2 | 教学主管 | 点击热力图格子能查看该时段所有学生明细 | 快速了解哪些学生安排了该时段的课程 |
| 3 | 行政人员 | 批量导出所有学生的课程日历图片 | 发放给学生和家长，方便他们查看课程安排 |
| 4 | 排课老师 | 系统自动检测排课异常（日期/每日限额/连续性） | 在导出前及时发现并修正排课错误 |
| 5 | 学生家长 | 收到清晰美观的课程日历（手机版或打印版） | 方便查看孩子的课程安排 |

## 三、需求池

### P0（核心功能，必须完成）

#### P0-1: 排课密度热力图
**描述**: 以热力图形式展示每周排课密度，纵轴为周一至周日，横轴为30分钟时间桶。

**验收标准**:
- [ ] 纵轴固定显示周一~周日（7行）
- [ ] 横轴按30分钟时间桶自动从数据收集并排序
- [ ] 时间桶计算规则：按课程开始时间向上取整归并（例如 9:15 → 9:30 桶）
- [ ] 聚合逻辑：GROUP BY (星期, 时间桶) → COUNT(学生)，同一学生同一时段同一讲次去重
- [ ] 颜色映射：动态线性映射（HSL 色相210，亮度随 opacity 变化）
- [ ] 格子内显示具体人数
- [ ] 文字颜色根据背景亮度自动切换（亮底深色字 / 暗底白色字）
- [ ] 右上角显示图例：□0人 □低 □中 □高 □最高N人
- [ ] 点击格子可下钻查看该格子的学生明细

#### P0-2: 下钻明细表格
**描述**: 点击热力图格子后，底部展开该格子的学生列表，支持排序和导出。

**验收标准**:
- [ ] 点击热力图某格子 → 底部展开该格的学生列表
- [ ] 表格列：学员名称、课程名称、课节（第X讲）、约课时间、在线状态
- [ ] 支持三态排序（点击列头循环切换：升序→降序→取消排序）
  - 学员名称：中文拼音排序（localeCompare 'zh-CN'）
  - 课节：提取数字按数值排（"第2讲"<"第10讲"）
  - 约课时间：日期排序
  - 在线状态：在线 > 离线
- [ ] 支持导出 CSV：表格右上角加「📥 导出本格 CSV」按钮
- [ ] CSV 列：姓名,课程名称,课节,约课时间,在线状态

#### P0-3: 异常检测
**描述**: 自动检测排课异常并在热力图上方展示异常列表。

**验收标准**:
- [ ] **排课日期异常**：学生的暑假课应该是11节课，应该放在某个阶段内（阶段日期写死在代码里），如果不在阶段内就报异常
- [ ] **每日排课异常**：学生每天排课不应超过1节，如果同一天超过1节排课就报异常（同一节课在1天内排了2次→按最新那个为主，重约功能，不是异常）
- [ ] **排课连续性异常**：学生排课应该是连续的（每周周二周三不排课），应该是每周的周1/4/5/6/7排课，如果不是这个逻辑就报错
- [ ] 异常展示：在热力图上方展示异常列表，每条异常注明：学生名 + 异常类型 + 详情说明

## 五、异常检测逻辑详细说明

### 5.1 排课日期异常

**判定规则**:
- 学生的暑假课应该是11节课
- 暑假课日期应该落在指定的阶段日期范围内：**2026-07-10 ~ 2026-09-04**
- 如果有任何一节暑假课不在阶段日期范围内，则报异常
- 通过课程名称包含「暑假」来判定是否为暑假课

**异常展示格式**:
```
🔴 {学生名} - 排课日期异常 - 课程"{课程名称}"安排在 {日期}，不在暑假阶段内（2026-07-10 ~ 2026-09-04）
```

**示例**:
```
🔴 张三 - 排课日期异常 - 课程"暑假班第3讲"安排在 2026-06-10，不在暑假阶段内
```

**备注**: 当前只检测暑假课的日期异常，后续可通过配置扩展其他课程类型的日期检测。

---

### 5.2 每日排课异常

**判定规则**:
- 学生每天排课不应超过1节
- 如果同一天超过1节排课，则报异常
- **例外情况**：同一节课在1天内排了2次 → 按最新那个为主（重约功能），不是异常

**异常展示格式**:
```
🟡 {学生名} - 每日排课异常 - {日期} 安排了{N}节课
```

**示例**:
```
🟡 李四 - 每日排课异常 - 2026-06-15 安排了2节课
```

**处理逻辑**:
- 检测同一学生同一天的所有排课
- 如果数量 > 1，检查是否为同一节课的重约（比较课程ID/名称）
- 如果是重约，保留最新记录，不报异常
- 如果是不同课程，报异常

---

### 5.3 排课连续性异常

**判定规则**:
- 学生排课应该是连续的
- **允许排课的日子**：周一、周四、周五、周六、周日
- **不允许排课的日子**：周二、周三
- **连续性定义**：排课日期应按周连续分布。例如：本周一/四/五/六/日 和 下周一/四/五/六/日 都有课，即为连续
- 如果排课日期出现在周二或周三，或者排课不连续（例如：本周有课，下周没有课），则报错

**异常展示格式**:
```
🔵 {学生名} - 排课连续性异常 - {日期}({星期})安排了课程，违反排课连续性规则
```

**示例**:
```
🔵 王五 - 排课连续性异常 - 2026-06-16(周二)安排了课程，违反排课连续性规则
🔵 赵六 - 排课连续性异常 - 本周有课但下周无课，排课不连续
```

**排课规则说明**:
- ✅ 允许排课：周一、周四、周五、周六、周日
- ❌ 不允许排课：周二、周三
- ✅ 连续性要求：按周连续排课（本周和下周对应日期都有课）

---

### 5.4 异常展示位置

**位置**: 热力图上方，使用醒目的卡片式展示

**样式**:
- 背景色：浅红色（#FFF0F0）
- 边框：红色左边框（4px solid #FF4444）
- 图标：根据异常类型使用不同颜色（🔴日期异常 🟡每日异常 🔵连续性异常）
- 可折叠：默认展开，用户可点击收起

**交互**:
- 点击某条异常 → 定位到热力图对应位置（高亮显示）
- 鼠标悬停 → 显示修复建议 tooltip
- 每条异常旁边有「忽略此异常」按钮（用于误报情况）

### 7.2 时间桶计算
```javascript
function getTimeBucket(startTime) {
  const [hours, minutes] = startTime.split(':').map(Number);
  const totalMinutes = hours * 60 + minutes;
  const bucketMinutes = Math.ceil(totalMinutes / 30) * 30;
  const bucketHours = Math.floor(bucketMinutes / 60);
  const bucketMins = bucketMinutes % 60;
  return `${String(bucketHours).padStart(2, '0')}:${String(bucketMins).padStart(2, '0')}`;
}
```
