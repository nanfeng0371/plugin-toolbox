## 一、产品定义

### 1.1 一句话描述

**辅导老师每日工作的可视化指挥中心**：自动拉取排课与学情数据 → 按 7 类规则自动归类 → 逐项打勾完成 → 实时同步到管理看板。

### 1.2 核心价值

| 角色 | 痛点 | 解决 |
|------|------|------|
| 辅导老师 | 每天手动核对 Excel，靠记忆跟进学生，容易遗漏 | 插件自动分类，逐项打勾，不遗漏 |
| 管理 | 不知道团队每个人做到哪了，只能靠口头汇报 | 云端看板实时查看，数据驱动管理 |

### 1.3 产品形态

- **辅导老师端**：Chrome 浏览器扩展（插件工作箱子模块），在爱芯后台页面使用
- **管理端**：EduFlow 管理后台 Web 页面
- **数据层**：CloudBase 云开发（NoSQL 数据库 + 静态托管）

## 二、用户角色与场景

### 2.1 角色定义

| 角色 | 标识方式 | 权限 |
|------|---------|------|
| 辅导老师 | `teacherName`（从爱芯后台页面提取当前登录用户名） | 仅读写自己的数据 |
| 年级组长 | `teacherName` + 角色标记 | 只读管辖团队数据 |
| 运营负责人 | `teacherName` + 角色标记 | 只读全量数据 |

> **决策记录**：一期用姓名做唯一标识（当前无重名），预留 `teacherId` 字段，未来可扩展为"姓名+工号"。

### 2.2 典型场景

| 场景 | 时间 | 动作 |
|------|------|------|
| 早上到岗 | 9:00 | 打开爱芯后台 → 打开插件 → 点击刷新 → 看到今日所有学生分类 |
| 上课期间 | 9:00-18:00 | 逐类完成辅导动作，每完成一个点一下 ✓ |
| 课后跟踪 | 下课后 30min | 刷新 → 看到"未生成报告"名单 → 联系学生重约 → 打勾 |
| 下班收尾 | 18:00 | 确保所有 7 类学生都有对应动作完成 |
| 管理查看 | 随时 | 打开 EduFlow 看板 → 看团队整体情况 → 发现问题及时介入 |

## 四、功能规格

### 4.1 Module A：学情表绑定

**触发**：首次使用，或学情表变更时

**入口**：侧边栏「📋 每日看板」Tab → 「📎 绑定学情表」

**输入格式**（Tab 分隔，从 Excel 复制）：
```
学生ID		学生姓名	手机号	年级	中心	...其他字段
10001	张三	13800001111	初一	天河	...
```

> **studentId 格式要求**：
> - 学情表首列必须是**纯数字** studentId（如 `10001`）
> - 爱芯排课 API 返回的 `studentId` 也是纯数字格式，两者可直接匹配
> - 如果格式不一致（如学情表是字符串"STU001"但 API 返回数字 10001），匹配会失败，粘贴时会提示"以下学生未在排课API中找到匹配"
> - 建议：学情表首列设置为「爱芯平台学生ID」（从爱芯后台导出）

**存储**：`chrome.storage.local`，key 为 `dailyboard_student_info`

**匹配逻辑**：
1. 解析粘贴文本，按 Tab 拆分为 `{ studentId, studentName, phone, grade, center, ... }`
2. 排课 API 返回数据中提取 `studentId`
3. 以 `studentId` 为 key 左连接学情表
4. 匹配不上的学生单独列出，提示"未在学情表中找到"

**交互细节**：
- 粘贴区有模板提示："从 Excel 复制学生信息后粘贴到此处（Tab 分隔）"
- 粘贴后自动解析并显示"已绑定 N 名学生"
- 支持重新粘贴覆盖

### 4.2 Module B：7 类学生自动分类

#### 4.2.1 数据源策略（v2 修正）

| 数据 | Phase 1 来源 | Phase 2 增强 |
|------|-------------|-------------|
| 学生排课信息 | `next/class/list` API | 同 |
| 是否有报告 | API 的 `reportVersion` 字段（0/1） | 同 |
| 掌握度等级 | API 的 `userPeriodLevel` 字段（A/B/C） | 同 |
| 四维评价标签（⭐优秀等） | **Phase 2** | 复用报告模块获取完整评价 |

> **v2 说明**：Phase 1 用爱芯排课 API 自带的 `reportVersion` 和 `userPeriodLevel` 做分类，不调报告 API。Phase 2 再通过现有报告模块获取完整四维评价标签。

#### 4.2.2 分类规则（v2 修正版决策树）

```
查询今日排课（所有学生）
        │
        ├─ 今日有排课？
        │   │
        │   ├─ 否 → 【分类7】今天没课
        │   │
        │   └─ 是 → 判断时间
        │       │
        │       ├─ 当前时间 < 排课开始时间
        │       │   └─ 【分类1】今天有课-未上课
        │       │
        │       ├─ 排课开始时间 ≤ 当前时间 < 排课结束时间
        │       │   └─ 【分类2】今天有课-正在上课
        │       │
        │       └─ 当前时间 > 排课结束时间 → 已下课
        │           │
        │           ├─ 无报告 + 下课 < 30分钟 → 等待报告（暂入分类2+标签）
        │           ├─ 无报告 + 下课 ≥ 30分钟 → 【分类3】已下课-无报告
        │           │
        │           ├─ 有报告 + 掌握度 A/A+ → 【分类4】已下课-表现好
        │           ├─ 有报告 + 掌握度 B/B+ → 【分类5】已下课-一般
        │           └─ 有报告 + 掌握度 C/无  → 【分类6】已下课-需跟进
        │
        └─ 学情表有但排课无 → 【分类7】今天没课
```

**关键修正**：
- 下课 < 30 分钟无报告 → **不判为分类 3**，先显示"⏳等待报告"标签
- Phase 1 分类 4/5/6 用 `userPeriodLevel`（A/B/C）判定，Phase 2 替换为完整四维标签

#### 4.2.3 分类 Tab 排序与辅导动作

| # | 分类 | 排序 | 辅导动作 | 打勾效果 |
|---|------|------|---------|---------|
| 1 | 今天有课-未上课 | 1 | 无（等待上课） | **无 checkbox** · 不计入完成率 |
| 2 | 今天有课-正在上课 | 2 | 巡课观察 | **无 checkbox** · 不计入完成率 |
| 3 | 已下课-无报告 | 3 | 电话联系 → 重约课 | ☐ 勾选 + 确认是否已重约 |
| 4 | 已下课-表现好 | 4 | 要笔记 / 抽查 / 完整复述 | ☐ 勾选即完成 |
| 5 | 已下课-一般 | 5 | 抽查 / 完整复述 | ☐ 勾选即完成 |
| 6 | 已下课-需跟进 | 6 | 电话联系家长 | ☐ 勾选即完成 |
| 7 | 今天没课 | 7 | **私聊布置题目** | ☐ 勾选即完成 |

#### 4.2.4 「未生成报告」特殊处理（v2 修正）

- 允许延迟：课后 **30 分钟**内未生成报告属于正常等待
- 下课 < 30 分钟 + 无报告 → 暂显示"⏳等待报告"，不归入分类 3
- **下课 ≥ 30 分钟**仍无报告 → 归入「分类 3」
- 勾选时多一步：弹窗确认"是否已帮学生重约？"
  - 选 ✓已重约 → 记录 `rebooked: true`
  - 选 ✗未重约 → 记录 `rebooked: false`，保持高亮提醒

### 4.4 Module D：云端同步

#### 4.4.1 同步时机与打勾状态保护

| 时机 | 方式 | 打勾状态处理 |
|------|------|--------------|
| 辅导老师打勾/取消 | 增量上传 | 只更新该学生 `isDone` 字段 |
| 点击「刷新数据」 | **分类重算 + 保留打勾** | 重新拉 API → 重分类 → **已打勾的学生保留 `isDone: true`（按 studentId 匹配） |
| 侧边栏关闭 | 全量上传 | 确保最新状态同步云端 |

**刷新时打勾状态保护逻辑（伪代码）**：
```javascript
// 刷新前：保存当前打勾状态
const savedDone = {};
currentStudents.forEach(s => savedDone[s.studentId] = s.isDone);

// 刷新后：重分类得到 newStudents
newStudents.forEach(s => {
  if (savedDone[s.studentId]) {
    s.isDone = true;  // 保留打勾状态
  }
});
```

> **关键原则**：刷新只改变「分类归属」，不改变「已完成/未完成」状态。除非该学生今天不再有课（从分类3移入分类7），则打勾状态清零（因为动作变了）。

### 4.6 教师设置与权限（v1.3 新增）

#### 4.6.1 插件端教师设置

辅导老师首次使用每日看板时，需要手动设置以下信息（保存到 `chrome.storage.local`）：

| 设置项 | 存储 key | 说明 |
|--------|---------|------|
| 教师姓名 | `db_teacherName` | 自动从爱芯 `bizInfo` 接口获取 |
| 所属学科 | `db_teacherSubject` | **手动选择**：数学 / 英语 / 物理 / 化学 等 |
| 所属年级 | `db_teacherGrade` | **手动选择**：初一 / 初二 / 初三 / 高一 等 |
| 所属中心 | `db_teacherCenter` | **手动选择**：天河 / 越秀 / 海珠 等（可选） |

> **设计理由**：`teacherSubject`/`teacherGrade` 是 CloudBase 管理看板按维度筛选的关键字段。爱芯 API 不提供教师的学科/年级信息，因此由辅导老师在插件设置中手动选择一次即可（存储在 local storage 中，后续自动复用）。
>
> **限制**：当前版本一个教师只教一个年级（单值 `string`），不支持多年级。若未来需要，可升级为数组。

## 六、数据模型

### 6.1 核心文档：teacher_daily_tasks

```typescript
// CloudBase NoSQL 文档结构
interface DailyTaskDoc {
  _id: string                    // 自动生成
  teacherName: string            // 辅导老师姓名（主键维度）
  teacherJobNumber: string      // 工号
  teacherCenter: string         // 中心（已有）
  teacherSubject: string        // 🆕 方案 A：学科，便于 EduFlow 按学科筛选
  teacherGrade: string          // 🆕 方案 A：年级，便于 EduFlow 按年级分组
  date: string                   // "2026-06-13"（主键维度）
  lastSyncTime: string           // ISO 8601 最后同步时间
  students: StudentTask[]        // 学生任务列表
  summary: TaskSummary           // 汇总
  createdAt: string              // 首次创建时间
  updatedAt: string              // 最后更新时间
}

interface StudentTask {
  studentId: string              // 学生ID
  studentName: string            // 学生姓名
  phone: string                  // 手机号（来自学情表）
  grade: string                  // 年级（来自学情表）
  center: string                 // 中心（来自学情表）
  courseName: string             // 课程名称
  courseStartTime: string        // ISO 8601 上课开始时间
  courseEndTime: string          // ISO 8601 上课结束时间
  category: number               // 1-7 分类编号
  categoryLabel: string          // 分类中文名
  isDone: boolean                // 是否已打勾
  doneTime: string | null        // 打勾时间 ISO 8601
  rebooked: boolean | null       // 仅第3类：是否已重约
  reportLevel: string | null     // 仅第4-6类：报告四维标签
  lessonStatus: string           // "upcoming" | "ongoing" | "finished"
  extra: Record<string, any>     // 扩展字段（预留）
  // 🆕 方案 A：冗余字段，便于 EduFlow 按维度分组
  teacherSubject?: string        // 如 "数学"
  teacherGrade?: string         // 如 "初一"
}

interface TaskSummary {
  totalWithCourse: number        // 今日有课总人数
  totalNoCourse: number          // 今日没课人数
  needActionCount: number        // 🆕 需要打勾的总人数 = 分类3+4+5+6+7
  categoryCount: {               // 各分类人数
    "1": number  // 未上课（无 checkbox）
    "2": number  // 正在上课（无 checkbox）
    "3": number  // 无报告
    "4": number  // 表现好
    "5": number  // 一般
    "6": number  // 需跟进
    "7": number  // 没课
  }
  doneCount: number              // 已完成总数
  doneRate: number               // 完成率 = doneCount / needActionCount（不含分类1/2）
}
```

### 6.3 数据保留策略 🆕

| 参数 | 值 | 说明 |
|------|-----|------|
| 数据保留期限 | **150 天** | 超过 150 天的 `teacher_daily_tasks` 文档自动清理 |
| 清理方式 | Phase 1 手动 / Phase 2 云函数定时 | CloudBase 定时触发器每天凌晨执行 |
| 保留目的 | 学期复盘 + 数据回溯 | 150 天覆盖完整学期（~5 个月） |
| 清理粒度 | 按 `date` 字段 | `date < today - 150` 的文档删除 |

> **决策记录**：原始建议 90 天，用户反馈不够（学期约 5 个月），延长到 150 天。CloudBase NoSQL 免费额度 2GB，按统一估算（见 6.4），150 天约 225MB，完全在免费额度内。

### 6.4 数据量估算（v1.3 统一校准）

| 参数 | 值 |
|------|-----|
| 辅导老师数 | ~10 人 |
| 每老师每天学生数 | ~150 人 |
| 每个学生文档字段大小 | ~1 KB（含 className/scheduleTime/isDone/tags 等） |
| 每老师每天文档大小 | ~150 KB |
| 每天总数据量 | ~1.5 MB |
| 150 天数据量 | **~225 MB** |

→ CloudBase 免费额度 2GB 充足，150 天仅占 11%。

> 所有接口均由 content.js 直接 fetch（共享页面 cookie，同源请求无 CORS 问题）。

| API | 真实 URL 路径 | 用途 | 请求方式 |
|-----|--------------|------|---------|
| 教师信息 | `GET /authorization/api/user/bizInfo` | 获取当前登录教师姓名（从页面 cookie 提取） | GET |

**接口调用示例（content.js）**：
```javascript
// 1. 获取教师姓名（从 bizInfo 接口）
const bizRes = await fetch('/authorization/api/user/bizInfo', {
  credentials: 'include'
});
const bizData = await bizRes.json();
// 返回格式：{ code: 0, data: { name: '甘海凤', ... } }

// 2. 获取今日排课
const scheduleRes = await fetch('/regularCourse/next/class/list', {
  method: 'POST',
  credentials: 'include',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    classStatus: '1',              // 进行中
    courseClassify: 3,             // 课程类型
    operationType: 1,
    current: 1,
    size: 50
  })
});
```

> **注意**：`getLoginParam` 不是真实接口名，已删除此引用。教师姓名从 `bizInfo` 接口获取。
