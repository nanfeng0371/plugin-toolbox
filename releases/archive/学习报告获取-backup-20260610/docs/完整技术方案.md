# 学习报告批量获取与分析 — 完整技术方案

> 最后更新：2026-05-29

---

## 一、项目背景

- **目标**：批量获取200个学生的学习报告数据，自动分析听课质量，输出Excel汇总
- **平台**：AI老师（aitutor100）+ 内部工作台（yuaiweiwu.com）
- **核心场景**：一对一互动课，学生应该高频回答老师提问，回答率是第一判断标准

---

## 二、完整API链路（4步）

### Step 1：获取学生列表

```
GET https://ai-genesis.yuaiweiwu.com/prod-api/student-center-ai/regularCourse/next/class/list
    ?classStatus=2                                    # 2=已结束
    &startDate=2026-05-24 00:00:00                    # 开始日期
    &endDate=2026-06-30 00:00:00                      # 结束日期

Headers:
    Cookie: authorization-token={JWT_TOKEN}

Response → data.classList[]:
    每个元素包含:
    - studentId       (如 "443504")
    - studentName     (如 "珂源")
    - periodId        (如 "64431")        ← 关键ID
    - aiPeriodId      (如 "11141")
    - courseClassify  = 3                ← 课程类型
    - studyVersion    = 1                ← 版本号
    ⚠️ 注意：列表里没有 report token！需要下一步生成
```

### Step 2：生成报告短链（关键发现🎯）

```
GET https://ai-genesis.yuaiweiwu.com/prod-api/student-center-ai/ai/teacher/ai/biz
    ?id={periodId}           ← 用上一步拿到的periodId
    &urlType=2               # 固定值，报告链接类型
    &broadcastType=4         # 固定值
    &courseClassify=3        # 固定值，同列表接口

Headers:
    Cookie: authorization-token={JWT_TOKEN}

Response:
{
    "code": "000000",
    "data": {
        "aiBizUrl": "https://s1.aiv5.cc/2FUR4E"   ← 短链
    }
}
```

### Step 3：短链重定向 → 提取report token

```
请求: GET https://s1.aiv5.cc/2FUR4E
响应: 302 重定向 → Location header 包含:
      https://next.aitutor100.com/reportV2.html?report={REPORT_TOKEN}&courseType=3&studyVersion=1

从Location中正则提取: report=(.+?)(&|$)
```

**或者直接拼接**（如果重定向格式固定）：
```
report token + courseType=3 + studyVersion=1
```

### Step 4：获取学习报告数据

```
GET https://next.aitutor100.com/ai-math-engine/lesson/report/queryCoursePeriodReport
    ?report={REPORT_TOKEN}&courseType=3&studyVersion=1

Response → 完整JSON数据（约9KB），包含：
- 学生姓名、课程名、课节名、授课老师
- focusInfo: 专注度评级(A/B/C/D)、认真回答次数、超过同学比例%
- knowledgeDtoList[]: 知识点数组，每个含：
    - knowledgeName, masteryRating(A+/A/B+/B/C/D)
    - interactExample: teacherAsk, stuAnswer, firstCorrect, guideCorrect, guideNum, interactExampleNum
    - exerciseCount, exerciseCorrect, isCompleted
- courseDetail[]: 课堂组件明细，componentType==3 为练习题，含 answerCorrect(true/false)
- interactNum: 总互动次数
- teacherComment: 老师评语文本

可选（第二个接口）:
GET https://next.aitutor100.com/.../summary?uid={studentId}&periodId={periodId}
Response → mistakeSummaryVo: { wrongNum: 2, questionNum:9 }  错题本统计
```

## 三、认证方式

| 项目 | 详情 |
|------|------|
| 认证类型 | JWT Bearer Token |
| Cookie字段 | `authorization-token` |
| 来源域 | `yuaiweiwu.com`（工作台域名） |
| 有效期 | 约24小时（exp在JWT payload里） |
| 数据接口认证 | **不需要token**，只靠URL里的report参数 |
| **结论** | 只有Step1和Step2需要JWT；Step3和Step4完全无需认证 |

## 四、评价逻辑（四层递进，不可违反）

### 第一层：参与度判定（最关键，态度判断）

```
回答率% = 学生总回答数 ÷ 老师总提问数 × 100%

| 回答率范围 | 判定标签 | 含义 |
|-----------|----------|------|
| ≥80% | ✅积极互动 | 全程配合 |
| 60%-79% | 👍正常参与 | 基本达标 |
| 40%-59% | ⚠️不太积极 | 需关注 |
| 20%-39% | 🔴敷衍上课 | 态度问题 |
| <20% | 🚨严重敷衍 | 必须干预 |
```

> 一对一互动课的核心就是高频问答。老师问18次你只答5次（28%），剩下13次"自问自答" = 敷衍挂机。

### 第二层：学习效果判定（能力判断）

```
首次答对率% = 首次答对数 ÷ 学生总回答数 × 100%
引导后答对率% = 引导答对数 ÷ 引导提问总数 × 100%

| 掌握情况 | 判定标签 | 条件 |
|---------|----------|------|
| ✅掌握扎实 | 首次答对率≥60% 且 掌握度A/A+ | 独立解题能力强 |
| 👍基本掌握 | 首次答对率30-59% 或 掌握度B+ | 有一定基础 |
| ⚠️有漏洞 | 首次答对率10-29% 或 掌握度B/C | 需要补漏 |
| 🔴未掌握 | 首次答对率<10% 或 掌握度D | 基本没学会 |

⚠️ 注意：回答量<3次时，正确率统计意义有限，标签需加"?"标记
```

### 第三层：练习情况判定

```
练习正确率% = 做对数 ÷ 有记录练习数 × 100%

| 正确率 | 判定标签 |
|--------|----------|
| ≥80% | ✅全对 |
| 50-79% | 👍大部分对 |
| 20-49% | ⚠️错较多 |
| <20% | 🔴大部分错 |
| 无练习数据 | - |
```

### 第四层：四象限综合分类

```
          高正确率              低正确率
高参与度  Q1 ✅优秀            Q2 👍认真需辅导
低参与度  Q3 🟠异常(需核实)    Q4 🚨敷衍+未掌握

综合标签规则：
- Q1高参与+高正确 → ⭐优秀
- Q1高参与+中正确 → 👍认真
- Q2高参与+低正确 → ⚠️需关注（能力问题）
- Q3低参与+高正确 → 🟠异常（可能作弊或数据异常）
- Q4低参与+低正确  → ❌敷衍+未掌握 / 🚨敷衍预警
```

## 六、执行流程

```
┌─────────────────────────────────────────────┐
│  输入：用户的JWT Token（从Cookie获取）        │
├─────────────────────────────────────────────┤
│  Step1: 调列表接口 → 拿到所有学生的periodId   │
│  Step2: 循环调biz接口 → 每个学生拿到短链     │
│  Step3: 跟随重定向 → 提取report token        │
│  Step4: 调数据接口 → 拿到完整学习数据         │
│  Step5: 解析+计算 → 按四层逻辑打标            │
│  Step6: 写入Sheet1（学生明细）               │
│  Step7: 分组汇总 → 写入Sheet2（课节汇总）     │
│  Step8: 输出Excel                            │
└─────────────────────────────────────────────┘
```

预估耗时：200人约5-8分钟。

## 八、注意事项

- **知识点数量不固定**（3-6个），预留6组知识点块，多余的留空
- **summary接口可能部分学生无数据**，需容错处理
- **课堂练习两套口径**（知识点级 vs detail级），detail级可能有缺口
- **JWT有效期约24小时**，过期后需要重新从浏览器Cookie获取
- **内部工作台是异步加载**（JS动态渲染），不能直接爬HTML
- **rate limit注意**：x-ratelimit-limit=3500，200个请求不会触发限制
